关于VUE-cli的一些简单的使用(一)

2019-02-15
本文总阅读量次

VUE

在说vue-cli之前,我们先来聊一聊vue.

vue是什么?

是一套用于构建用户界面的渐进式框架.Vue可以用于自底向上逐层应用.其核心库只关注视图层,上手快,还便于整合第三方库或项目[这里就不做过多的描述,可以去看看官文]

Vue的核心思想:组件化和数据驱动.

什么是组件化?

组件化就是将一个整体合理的拆分为一个一个小块(这就是组件),组件可以重复使用.

数据驱动是前端的未来发展方向,释放了对DOM的操作,让dom随着数据的变化自然而然的变化(大佬说的,记下来),所以我们操作的就是数据

Vue-cli是快速构建单页面的脚手架.如果在此之前,你选择使用vue-cli来入门的话,就选择引入js文件, [不是别的意思,只是这里使用的东西很多,你没有使用过vue的话,很容易走上一条 (从还没入门到放弃)的道路]

然后,我们来构建一个很简单的项目吧

构建开始

首先,没有环境怎么用呢,所以我们要先搭建环境(这里默认你已经有了最基本的环境)

vue init webpack hello
`

hello是项目名称,可以自己定义

[如果你不会的话 ,可以一路回车下来] 等命令执行完成你就有一个项目了 {tips:你现在可以npm run dev 来启动这个项目}

现在你可以打开你的浏览器 在地址栏敲入http://localhost:8080 ,然后你就能看到你的界面了

然后我们回到这个项目里面来 打开你的项目

不出意外的话,你现在有的项目结构就像上图一样,

build 这个文件夹里面是一些操作文件,使用 npm run *时,就是去执行这里面的文件

config 配置文件,执行文件需要的配置信息

node_modules 这个文件夹里面,存放了你项目所有的依赖文件

src 资源文件,所有的组件以及所有用到的图片都是存放在这个里面的

static 这里面一般存放的是 引入的第三方静态文件,比如说ueditor之类的

文件夹说完了,就来说说下面主要 使用的文件了吧

首先就是package.json文件 打开这个

你就能看到当时你在安装的一些信息,如果你觉得你技术可以的话[你可以手写这个]

这里面的scripts里面 就是你可以执行的命令项npm run + 你要执行的什么语句块

而dependencies就是你的依赖项了

对了说到这里,关于编辑器的选择,[个人推荐一下vscode].只是感觉很好用,没有别的意思.
